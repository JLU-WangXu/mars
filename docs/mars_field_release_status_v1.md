# MARS-FIELD Release Status v1

## Recommended Public Status

This repository is ready to publish as:

**MARS-FIELD engineering approximation v1**

## What This Means

The codebase is already complete enough to share as a public research prototype because it includes:

- a unified engineering workflow
- target-level pipeline execution
- benchmark execution
- retrieval-aware evidence
- ancestry-aware evidence
- pairwise decoder logic
- calibrated selection
- reproducible paper-oriented figure generation

## What Is Functionally Complete

- single-target design pipeline
- benchmark runner
- family and held-out-family summaries
- motif-atlas retrieval branch
- ancestral field branch
- structured decoder branch
- calibrated selector / ranker
- paper bundle generation
- PyMOL/PSE structure rendering
- manuscript-oriented figure rendering

## What Is Not Final Yet

This repository should **not** be described as:

- the final fully neural end-to-end field model
- a finished production system
- the final algorithmic endpoint of the project

The biggest open gap remains:

- the fully learned neural field path is still only partially established
- the current public system is an engineering approximation rather than the terminal model form

## Recommended GitHub Framing

Use language like:

`MARS-FIELD is a benchmarked protein engineering research prototype that builds a shared residue decision field from retrieval, evolution, ancestry, and structure-aware evidence.`

Avoid language like:

`MARS-FIELD is a completed end-to-end neural field model.`

## Bottom Line

For GitHub:

- **yes**, the repository is complete enough to package and upload
- **no**, it should not be marketed as the final neural algorithm version

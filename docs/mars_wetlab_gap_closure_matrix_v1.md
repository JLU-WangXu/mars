# Mars Wet-Lab Gap Closure Matrix v1

## 1. 用途

这份文档不是讲算法，而是给实验同事直接看的。

目标只有一个：

- 让人一眼看懂现在还缺哪些实验
- 每个实验是为了验证什么
- 最小对照组是什么
- 什么结果出现了，才能说这一条线往前推进了

## 2. 总判断

当前最需要补的不是更多计算，而是三类实验闭环：

1. `Cld` 主线功能韧性闭环
2. `AresG` 保护模块功能闭环
3. `DrwH` / `AresW` cargo-cap 闭环

这三类里，优先级最高的是 `Cld`。

原因很简单：

- `Cld` 是主功能酶
- 它最适合承载“火星相关多压力条件下功能韧性”的主叙事
- 只要 `Cld` 数据起来，整条线就能从计算故事进入真正的生物学故事

## 3. Cld 主线：必须补的实验

### 3.1 第一组：表达与基础功能面板

**对象**

- `Q47CX0`
- `Node007`
- `Node012`
- `Node006`

**为什么做**

这组实验不是为了证明“火星相关性”，而是为了先回答最基础的问题：

- 祖先骨架能不能表达
- heme 能不能正确装配
- 基础 `chlorite` 活性还在不在

如果这一步都不过，后面的氧化、低温、干燥复水都没有意义。

**最小 readout**

- 可溶表达量
- 纯化后是否有 heme 装配信号
- 标准条件下基础 `chlorite` 活性

**最小对照**

- 天然体 `Q47CX0`

**成功标准**

- 至少 1 条祖先体满足：
  - 可溶表达 `>= 0.30 x WT`
  - 基础活性 `>= 0.50 x WT`

这一步一旦成立，就说明“祖先骨架路线”值得继续，而不是只剩概念。

### 3.2 第二组：氧化挑战后剩余活性

**对象**

- `Q47CX0`
- `Node007`
- `Node012`
- `Node006`

**为什么做**

这是把“抗辐照”翻译成实验上更可执行的蛋白层 readout。

这里真正要验证的不是“能不能扛宇宙射线”，而是：

- 在 ROS / 氧化打击后，酶活掉得更慢吗

**最小 readout**

- `H2O2` 挑战后剩余活性

**推荐扩展**

- 多个 `H2O2` 剂量
- 处理前后聚集/沉淀趋势

**最小对照**

- 天然体 `Q47CX0`
- 同批次空白 buffer 对照

**成功标准**

- 至少 1 条祖先体在氧化挑战后剩余活性高于天然体

这一步一旦成立，就说明主线已经开始超出“普通稳定性工程”。

### 3.3 第三组：低温功能面板

**对象**

- `Q47CX0`
- `Node007`
- `Node012`
- `Node006`

**为什么做**

火星相关主叙事里不能只有氧化，还必须有低温维度。

这里要验证的是：

- 在低温下功能还能不能保留
- 祖先骨架是不是比天然体更适合作为低温工程底盘

**最小 readout**

- `4 C`
- `10 C`
- `20/25 C` 作为标准条件对照

**最小对照**

- 天然体 `Q47CX0`

**成功标准**

- 至少 1 条祖先体在低温下保留更高相对活性

### 3.4 第四组：干燥-复水 / 冻融面板

**对象**

- `Q47CX0`
- `Node007`
- `Node012`
- `Node006`

**为什么做**

这一步是把“火星多压力”里的物理应激真正补上。

这里验证的不是结构漂不漂亮，而是：

- 经历物理损伤后，酶还能恢复多少功能

**最小 readout**

- `FT03`：3 次冻融后的剩余活性

**推荐扩展**

- `LYO01`：1 次干燥-复水后的剩余活性

**最小对照**

- 天然体 `Q47CX0`
- 同条件不处理对照

**成功标准**

- 至少 1 条祖先体在冻融或干燥复水后显著优于天然体

### 3.5 第五组：保守理性改造验证

**对象**

建议优先从：

- `Node007`
- `Node007 + W67Y`
- `Node007 + H271Q`
- `Node007 + W67Y + H271Q`

开始

**为什么做**

这一步不是再证明 ASR，而是证明：

- 祖先骨架确实更适合承载后续抗氧化理性改造

**最小 readout**

- 基础活性
- `H2O2` 后剩余活性
- 低温活性

**最小对照**

- `Node007`
- `Q47CX0`

**成功标准**

- 至少 1 条小突变版本在不明显损失基础活性的前提下，提高氧化后剩余活性

## 4. AresG 线：必须补的实验

### 4.1 第一组：表达 / 可溶性 / 基本稳定性筛选

**对象**

- `AresG2-01_shortlink_shorttail`
- `AresG2-02_midlink_balanced`
- `AresG2-03_longlink_longtail`

**为什么做**

现在 `AresG` 最大的不确定性不是概念，而是：

- 哪个版本不是“纸面上有趣”，而是真的能做出来

**最小 readout**

- 表达量
- 可溶性
- 是否明显聚集

**最小对照**

- `AresC-00_core_only`

**成功标准**

- 至少筛出 1 个主推版本和 1 个保留对照版本

### 4.2 第二组：应激保护功能验证

**对象**

- 从上一步筛出的 1 到 2 个 `AresG2` 版本

**为什么做**

这是 `AresG` 这条线最核心的问题：

- 它到底是不是保护模块

**最小场景**

- 冻干后 cargo 活性恢复
- 高渗/脱水后 cargo 活性恢复
- 低温处理后 cargo 活性恢复

**推荐 cargo**

- 先选一个好测、便宜、可重复的酶或报告蛋白

**最小对照**

- cargo 单独
- `AresC core-only`
- 传统赋形剂体系

**成功标准**

- 至少 1 个 `AresG2` 版本在 1 个应激场景中优于“cargo 单独”和“core-only”

## 5. DrwH / AresW 线：必须补的实验

### 5.1 第一组：祖先域本身的结构与表达筛选

**对象**

- `DrwHAnc-01_local_near_Node046`
- `DrwHAnc-02_local_mid_Node054`
- `DrwHAnc-03_deep_ingroup_Node068`

**为什么做**

这里不是先证明它保护 cargo，而是先回答：

- 哪条祖先域最像稳定紧凑的保护帽

**最小 readout**

- 表达
- 可溶性
- 紧凑性/折叠趋势

**最小对照**

- 天然 `Q9RUL2`

**成功标准**

- 选出 1 到 2 条最值得继续做 `cargo-cap` 的祖先域

### 5.2 第二组：AresW 拼接体验证

**对象**

- `AresW-03_C_cap_anc_local`
- `AresW-04_C_cap_anc_mid`
- `AresW-05_C_cap_anc_deep`

**为什么做**

这一步验证的是：

- cap 接到 core 上后，有没有把底盘拖坏
- 哪个 cap 更适合继续推进

**最小 readout**

- 表达
- 可溶性
- 是否明显拖塌 core

**最小对照**

- `AresW-00_core_only`

**成功标准**

- 至少有 1 个拼接体在不明显破坏 core 的前提下通过表达筛选

### 5.3 第三组：cargo-cap 保护功能实验

**对象**

- 上一步筛出的 1 个最优 `AresW` 或 `DrwH` 版本

**为什么做**

这是 `DrwH` 线真正成立与否的决定性实验：

- 它到底能不能保护 cargo

**最小场景**

- 氧化后 cargo 活性保留
- 干燥-复水后 cargo 活性保留

**最小对照**

- cargo 单独
- core-only
- cap-only

**成功标准**

- 至少 1 个条件下，cargo-cap 方案显著优于 cargo 单独

## 6. 推荐给实验同事的最短执行顺序

如果实验资源有限，建议严格按下面顺序：

1. `Cld` 祖先表达 + 基础活性
2. `Cld` 氧化挑战
3. `Cld` 低温 / 冻融 / 干燥复水
4. `Cld` 小突变保守改造验证
5. `AresG2` 三版本表达筛选
6. `AresG2` cargo 保护实验
7. `DrwH` 祖先域表达筛选
8. `AresW` 拼接体验证
9. `DrwH/AresW` cargo-cap 保护实验

## 7. 一句话结论

实验上现在最该补的，不是更大的故事，而是：

- `Cld`：证明祖先骨架和保守改造真的提高多压力下的剩余功能
- `AresG`：证明它真的是保护模块，而不只是结构概念
- `DrwH`：证明它真的是 cargo-cap，而不只是小型稳定域

只要这三点里先把 `Cld` 做扎实，整条火星任务就会从“计算路线”进入“生物学结果”阶段。
